# Training and demonstration area - nonproduction

This area is strictly for fictitious projects, sample schedules, demonstrations, training scenarios, mock turnover packages, and synthetic records.

Controls:

- Never copy this data into production seed files, migrations, live navigation, reports, or analytics.
- Label every visible training screen and export as nonproduction.
- Use obviously fictitious organizations, people, project numbers, heat numbers, and signatures.
- Do not place customer-confidential records here.
- Use a separate identity tenant, database, storage location, and integration credentials from production.
- Test automated checks that prevent training/demo data from entering a production deployment.

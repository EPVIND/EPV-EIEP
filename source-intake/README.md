# Source intake - unapproved reference material

This area accepts prior prototypes, uploaded reference files, legacy forms, exports, screenshots, and other artifacts that may inform implementation. Nothing here is approved merely because it was transferred.

For each item:

1. Preserve the original filename and record provenance.
2. Record it in `docs/00-program/EXISTING_ASSETS_INDEX.md`.
3. Classify confidentiality, ownership, quality, currency, and production/demo status.
4. Scan for secrets, malware, personal data, customer restrictions, and fictitious records.
5. Compare it with controlled requirements and identify conflicts.
6. Copy only approved concepts or transformed assets into active areas; do not modify the intake original.
7. Record the approval and destination in the asset index.

Suggested routing:

- `existing-prototypes/`: application prototypes and code baselines.
- `uploaded-reference-files/`: customer-provided and project reference material.
- `legacy-forms/`: existing forms, logs, spreadsheets, and reports.
- `unreviewed/`: newly received items awaiting classification.

# Workflow: Welding

This workflow is a near-term extension and informs the shared data model.

## Prerequisites

- Current released drawing/isometric and joint identity.
- Traceable parent materials and filler/consumable lots as required.
- Approved WPS supported by applicable PQR and code/customer context.
- Welder/operator qualification valid for process, variables, continuity, and project requirements.
- Fit-up, preheat, environmental, cleanliness, purge, and hold-point requirements satisfied.

## Flow

1. Create/confirm weld identity linked to spool/member/component, materials, joint, system/line, drawing revision, and work package.
2. Assign WPS and verify welder/process eligibility at time of work.
3. Record fit-up and required inspection release.
4. Record weld process passes/participants, consumables, preheat/interpass, completion time, and production status at approved detail.
5. Complete visual examination and generate required NDE/PWHT/PMI/pressure-test dependencies.
6. Control repair cycle with defect/removal, approved repair basis, repair number, welder, re-examination, and final acceptance.
7. Include accepted records in system/test/turnover packages.

## Controls

No hard-coded code acceptance rules. Validity derives from controlled qualifications, procedures, project requirements, and qualified acceptance. Preserve original and repair history; never replace a failed examination with only the final passing result.

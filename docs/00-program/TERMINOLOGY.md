# Controlled Terminology

| Term | Meaning |
|---|---|
| EIEP | EPV Industrial Enterprise Platform. |
| dQMOS | Digital Quality Management Operating System; the governed quality backbone within EIEP. |
| Project | The top-level controlled delivery context for a customer scope. |
| Facility | A physical owner/operator site that may contain multiple projects. |
| System | A functional process, utility, electrical, structural, or commissioning boundary used for completion and turnover. |
| Area | A geographic or plant breakdown independent of system. |
| WBS | Work breakdown structure used for scope, schedule, cost, and responsibility alignment. |
| Work package | A controlled, executable scope with inputs, responsibilities, constraints, inspections, deliverables, and acceptance. |
| Performer | The organization or crew executing work: EPV, subcontractor, vendor, owner, or third party. |
| Material lot | A quantity received under common purchasing, specification, heat/lot, and certification attributes. |
| Material item | A uniquely traceable piece or controlled quantity derived from a material lot. |
| MTR | Material test report or material test certificate linked to the represented material. |
| PMI | Positive Material Identification used to verify alloy or material composition under an approved method. |
| Release | An authorized decision allowing a record, material, or work item to proceed to a defined next state. |
| Quarantine | A controlled status preventing use or issue pending resolution. |
| Hold point | Work cannot proceed without the designated party's release. |
| Witness point | The designated party is notified and may witness; waiver rules are project-controlled. |
| Review point | Documentation or evidence requires review without necessarily stopping physical work. |
| NCR | Nonconformance report controlling a departure from specified requirements. |
| Punch item | A completion deficiency or outstanding item tracked to verification and closure. |
| Turnover package | A governed set of accepted records delivered for a defined system, area, asset, test package, or contractual boundary. |
| Current revision | The approved revision authorized for the defined use and effective context. |
| Superseded | A historical revision retained for traceability but not authorized as current. |
| Production data | Real operational data in the production environment. It excludes training, demo, and synthetic records. |
| Source intake | Unapproved reference artifacts awaiting inventory and reconciliation. |
| Training/demo | Explicitly fictitious, isolated content used only for training, testing, and demonstration. |
